# Android M1 — unlock

- 日期：2026-07-22（Asia/Shanghai）
- 分支：`codex/android-m1-unlock`（基于 `main@a514117`）
- 契约：`docs/design/000` r16、`002`、`003` r15（规则 7）、`007`
- 结论：M1 验收项通过；未修改 Rust/UniFFI 模块接口。

## 交付

- 新增 Android 14+ Compose resident app，并直接持有一个进程内 `VaultSession`；UI、
  生物识别 Activity 与后续系统 service 共用该对象和核心 writer lock。
- 实现 Android Keystore AES-GCM unlock blob：auth-per-use `BIOMETRIC_STRONG`、
  `setInvalidatedByBiometricEnrollment(true)`、StrongBox 优先及 TEE 回退；密文与
  key alias/security level 作为 app-private `AtomicFile` 记录提交。
- blob 是单一逻辑原子项：新 key 先暂存、密文原子提交后才清旧 key；进程在任意一步
  退出时，启动调和会恢复 `.bak` 或清除未提交 `.new`/孤儿 key。撤销删除全部密文与
  `vaultkern.unlock.*` key；删除未完成会显式标为待重试。
- 失效 key、缺失 key、坏 tag 或不符合 release 硬件策略的旧 blob 都分类为
  `invalidated`，删除并在 UI 提示交互解锁后重新注册；用户取消和临时失败保留原项。
- 交互解锁后按 003 规则 7 调和注册；开关保存只提交 desired state，创建/撤销均在
  保存后的异步幂等调和以及每次成功解锁后执行。调和失败不会伪装成设置保存失败或
  解锁失败，并会在 UI 明示等待下一个调和点。
- 核心既有 unlock 路径负责文件 HMAC 验证、换盐 cache miss 时在 resident app 内
  重跑 KDF 并刷新 `{master credential, cached transformed key}`，以及移动 KDF 上限。
- Kotlin 明文 blob 缓冲在加/解密边界立即清零；credential 从 `CharArray` 通过可清零
  UTF-8 byte buffer 直接构造 UniFFI sensitive owner，不创建 Kotlin `String`，并在所有
  完成、错误和取消路径清零；UI state 的诊断输出固定遮蔽密码，窗口使用
  `FLAG_SECURE`，日志和 DTO 不承载 key material。

## 验证

设备：`VaultKern_M0_API34` AVD，Pixel 6 profile，Android 14 / API 34，
Google APIs x86_64。该 AVD 无 StrongBox，实际 Keystore security level 为 `SOFTWARE`；
debug 构建仅为模拟器验证允许 software，release 构建会拒绝并删除 software blob，
因此没有把本结果外推为物理设备 TEE/StrongBox 证明。

M0 的真实系统 ceremony 证据按任务要求直接沿用而未重跑：同一 AVD 上
BiometricPrompt AES-GCM round-trip 通过，新增指纹后 key 永久失效且 blob 删除。
M1 产品代码另用真实 `AndroidKeyStore` 的 `KeyInfo` 仪器测试确认 auth-per-use、
biometric-enrollment invalidation 标志与所记录安全级别一致。

执行结果：

- `cargo test --locked -p vaultkern-runtime app_unlock_hits_cache_then_refreshes_once_after_a_salt_change -- --nocapture`
  — 1 passed；证明文件 HMAC cache miss 后只刷新一次，随后命中新 cache。
- `cargo test --locked -p vaultkern-kdbx fixed_policy_threshold_matrix_covers_below_equal_and_above -- --nocapture`
  — 1 passed；证明 mobile Argon2 `>128 MiB` 与 AES-KDF `>600M` 拒绝。
- `cargo test --locked -p vaultkern-uniffi -- --test-threads=1` — 32 passed（4 suites）。
- `Gradle 8.7 --no-daemon :app:testDebugUnitTest` — 17 passed；覆盖 desired-state 提交顺序、
  幂等撤销/注册、调和失败分类、credential UTF-8 成功/异常路径清理和诊断遮蔽。
- `Gradle 8.7 --no-daemon :app:connectedDebugAndroidTest` — 16 passed；覆盖 resident core
  交互注册/biometric blob 解锁/撤销状态、跨保险库状态隔离、失效分类、取消保留、
  提交后清理故障、fresh UV、release 硬件策略、真实 Keystore key 属性及 Compose UI。
- `cargo build --locked --release -p vaultkern-uniffi` 分别对
  `x86_64-linux-android` 与 `aarch64-linux-android` 成功；Gradle debug/release APK
  均打入对应 ABI。
