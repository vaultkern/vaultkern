# Android M3 — credentials

- 日期：2026-07-23（Asia/Shanghai）
- 分支：`codex/android-m3-credentials`，堆叠在 M2 / PR #46
- 契约：`docs/design/000` r16、`002`、`003` r15、`007`
- 结论：M3 验收项通过；仅为复用既有 core URL/PSL 匹配新增一个小型、纯新增的
  `VaultSession.findFillCandidates` UniFFI 绑定，未改变 runtime 语义或既有接口。

## 交付

- `AutofillService` 与 Credential Manager `CredentialProviderService` 均未声明
  `android:process`，由 resident `VaultKernApplication` 直接复用同一个 `VaultSession`、
  writer lock、unlock adapter 与正常保存路径；没有 outbox 或 overlay 进程。
- Autofill 在 dataset 级别要求认证；fresh UV 后按系统请求来源调用核心既有 URL/PSL
  匹配，只揭示请求实际需要的 password / username / 当前 TOTP 并立即构造 OS Dataset。
  原生应用只匹配显式 `androidapp://package` 条目；WebView 必须通过调用包签名与站点的
  Digital Asset Links，受信浏览器则必须匹配固定 allowlist。跨协议的 `androidapp` 条目
  不会流入 Web，HTTPS 条目不会降级填入 HTTP。多候选时在 `FLAG_SECURE` Compose
  picker 中选择；取消、读取失败或不支持保存捕获时显式失败。
- Passkey 注册严格解析 WebAuthn creation options、执行 RP/origin 校验与 biometric UV，
  通过既有核心构造写入 KPEX-format entry，并在响应返回前走正常 durable save；若当前
  vault 来自用户选择的本地文档，注册还会复用 M2 的持久 save receipt 发布回同一个 URI，
  provider 暂时不可读/写时保留私有候选并由下次 reconciliation 重试。断言从核心列出
  候选、验证 allow-list/RP、由存储的私钥签名并返回核心 authenticator data。
- native 调用以当前签名证书计算 Android origin，并通过 Google Digital Asset Links API
  验证凭据共享专用的 `delegate_permission/common.get_login_creds`；浏览器调用必须匹配
  受信 allowlist，
  使用调用方提供的 32-byte `clientDataHash` 和规范允许的 placeholder `clientDataJSON`。
  allowlist 固定为 2026-07-22 Google 公共快照，SHA-256
  `594a83e3cb3475e8af8da22595b3481cbefb61b747c7a67a66aa2e47433ed79c`。
- 每个系统 credential entry 都绑定 request JSON 与 `clientDataHash` 的摘要，多个 option /
  challenge 并存时不会误签另一个请求。认证后选择超过 30 秒会重新要求 UV；失败或取消
  不刷新 freshness。注册提交失败会释放 M2 本地发布事务；断言完成后的临时 finish 失败
  允许再次清理，而正在完成时的 `onDestroy` 不会抢关。协程取消不会被转成普通 provider
  失败；注册、断言所有异常路径都会结束核心 operation，未提交注册回滚。
- 系统 provider 的启用仍由 Android 设置保存为用户期望状态；应用内没有“开关即执行”
  的旁路副作用。Kotlin 不接触 passkey 私钥；响应和诊断对象遮蔽 credential/TOTP/password，
  UniFFI sensitive owners 在 reveal 后立即关闭。Autofill 不再创建 password/TOTP DTO 或
  随 Dataset 返回的额外 value-map 容器，明文只保留到 OS Dataset 交接边界。

## 进程退出边界

- 注册在 core commit 前退出：磁盘不含新 passkey；若本地文档 receipt 已写，下次恢复会
  识别 private mirror 未变化并清理。commit 后、原 URI 发布前退出：已 durable-save 的
  KPEX entry 与 receipt 保留，下次启动继续发布；provider 已替换但 readback/清收据前退出，
  恢复会用内容指纹识别已完成写入，不重复制造冲突副本。系统在 response 前杀进程时，
  调用方仍可能看到取消/未知并重新发起 ceremony，这是 WebAuthn 外部边界而非旁路队列。
- 断言只读；任意退出都不创建旁路队列。核心 operation 在正常取消/错误时显式结束，
  进程被系统终止时随 resident process 一并释放。
